Strict HyperPBI 2.0

A compact authoring language that compiles to the existing renderer.

Version 2.0 rejects unknown properties, resolves Field Manifest aliases, scopes each component to one dataset, expands definitions and patterns, then validates the prepared component tree.

  • Stable IDs connect inspector edits, interactions, overlays, and state.
  • Root app defines brand, navbar, sidebar, page header, and footer.
  • Definitions reuse component fragments; arrays replace and objects deep-merge.
  • Patterns expand to normal components with deterministic child IDs.
  • Version 1.0 remains supported for existing dashboards without silent migration.

Fields and datasets

Model origin stays distinct from query aggregation and local calculations.

Sum(Sales.Amount) can still represent a summarized model column. A true model measure, dataset-derived field, and dataset metric have different interaction capabilities.

Field Manifest

AI alias, canonical key, display/query/source metadata, semantic role, data type, true measure versus implicit wrapper, and identity/filter eligibility.

Logical datasets

Named chains over the current Power BI data view: filter → derive → rename → select → group/metrics → distinct → sort → limit. Static schemas survive zero rows.

Dataset lineage

Filtered, grouped, and distinct output rows retain contributing source indices. Identity selection can follow lineage; derived metrics do not become model-filter targets.

Analysis and presentation

First-class semantics before escape hatches.

C

Semantic charts

Field bindings generate series data. Safe options can style presentation but cannot replace semantic data, axes, links, nodes, transforms, or series counts.

T

Native tables

Detail table and matrix support bounded rows, formatting, selection, pagination, density, actions, and empty states. Tabulator is compatibility input only.

M

Maps and ArcGIS

Power BI geometry, public feature services/joins, tile/basic dynamic layers, renderer/labels, layer panel, legend, search, and selection within package policy.

S

Governed SVG

Allowlisted elements, data contexts, bindings, bounded repeats, normal interactions, animation presets, local-ID isolation, exact limits, and reduced motion.

I

Three interaction systems

uiAction for interface state, universal interaction for data behavior, and safe event-specific interactions.

R

Visual Inspector

Select rendered components by stable ID, edit canonical fragments, validate the whole dashboard, and keep unrelated behavior intact.

Security boundaries

Declarative flexibility without an executable-content sandbox.

Every advanced surface has its own parser, allowlist, scope, and limit. Dashboard JSON never gains code execution or unrestricted network access.

JS

No user JavaScript

No eval, functions, callbacks, inline handlers, scripts, iframes, SQL, or network datasets.

H

Sanitized HTML and CSS

DOMPurify plus parsed, property-allowlisted, selector-scoped CSS. External imports, unsafe URLs, fixed positioning, and abusive z-index are blocked.

E

Safe chart options

Functions, URL-bearing keys, executable strings, prototype keys, and semantic data overrides are removed.

V

SVG isolation

Element/attribute/reference allowlists, namespaced IDs, no external resources, and exact component/dashboard budgets.

P

Prompt privacy

Samples, masked, summary, fields, or types modes. Prompts remain local until the user copies them.

W

Provider policy

Core has no WebAccess. Maps still requires HTTPS, no embedded credentials, package permission, runtime host policy, and privacy acknowledgment.

Distribution profiles

Choose network privilege explicitly.

Package profiles change external map/provider privilege, not the dashboard security model.

Core No WebAccess

Bound Power BI geometry and all non-network dashboard features. External tiles, services, and geocoders are unavailable.

npm run package:core

Maps Optional WebAccess

Broad https://* by default or restricted built-in/configured hosts. Public HTTPS ArcGIS/provider access only; no credentials in JSON.

npm run package:maps

Artifacts are packager-derived and labeled *-core.pbiviz, *-maps-broad.pbiviz, or *-maps-restricted.pbiviz, with a Maps compatibility copy.
