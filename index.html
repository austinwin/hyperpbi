AI authoring and response repair

HyperPBI compiles a self-contained instruction from the current fields, selected privacy mode, visual dimensions, requested components, calculations, interactions, map policy, and security boundaries.

Open AI guide

Field dictionary only

Default. Sends normalized keys, labels, lineage, type, format, and roles without row values.

Samples or masked samples

Use up to 50 selected rows. Masked mode replaces text/date values and zeroes numeric values.

Summary or types only

Share numeric ranges, date ranges, categories, or only inferred types when raw examples are unnecessary.

!
Review before sharing. HyperPBI never calls an AI API. The user explicitly copies the prompt to ChatGPT, DeepSeek, Copilot, or an approved internal model.

Generation contract

  • Return one JSON object with version:"1.0" and components.
  • Use normalized field keys only and give every component a unique ASCII id.
  • Prefer compact enterprise layouts, bounded tables, restrained colors, and normal components first.
  • Never return JavaScript, functions, scripts, iframes, callbacks, URLs, or invented fields.

Importer and repair

  • Accepts surrounding prose, Markdown fences, smart quotes, trailing commas, and export packages.
  • Repairs syntax only; schema, calculation, and field-reference validation still run.
  • Copy repair prompt includes the broken JSON, precise errors, fields, and supported contract.
  • The original saved dashboard is never replaced when validation fails.

Specification and Runtime Config

The Specification owns the visible application. Runtime Config owns renderer behavior, global interaction defaults, security modes, providers, geocoder settings, and map bindings.

Open specification

Specification roots

version, title, theme, layout, state, panels, components, calculations, styles, and css.

Shared component fields

type, id, title, span, className, props, style, css, slots, data, and interaction.

GUI-first Runtime Config

Form changes update JSON immediately. Advanced JSON remains a draft until Apply JSON; invalid JSON never mutates the applied GUI.

Universal interactions

Every component resolves the same policy. Internal HyperPBI behavior and external Power BI behavior are independent and can be combined deliberately.

internalScopeself, others, allDetermines which HyperPBI components receive the internal filter/highlight.
externalModenone, auto, selection, filterControls Power BI propagation. Auto filters controls and selects exact data identities.

Use selection for data points

Table rows, matrix cells, chart points, map features, and timeline items use exact Power BI selection identities in auto mode.

Use filters for field values

Controls and slicers use applyJsonFilter. Explicit filter mode requires a model column with sourceTable and sourceColumn.

Custom components, repeat templates, and slots

Custom content is expressive without becoming an arbitrary web runtime. HyperPBI owns wrappers, keyboard behavior, state, selection, sanitization, and event execution.

Open custom guide

Safe templates

Use tokens such as {{row.field}}, metrics, field metadata, props, state, and selected values.

Repeat and slots

repeat supports distinct, sorting, limit, and row templates. Slots include header, actions, row, cell, popup, tooltip, legend, and badge.

Typed actions

selectRow, selectWhere, filters, state changes, tabs, collapse, detail, highlight, and explicit clears.

Calculation DSL

Derived fields execute row-by-row in dependency order. Metrics execute over the current filtered row set. Invalid dependencies, unknown operators, and cycles are rejected before rendering.

Open DSL reference

Expression nodes

Field references use {"field":"risk_score"}; literals use {"value":80}.

Operator families

Arithmetic, comparison, boolean, text, date, null, conditional, and case operations.

Metric families

Count, conditional count, sum/average, min/max, distinct count, ratio, and percent of total.

Derived risk band and metric
{
  "fields": [{
    "key": "risk_band",
    "type": "text",
    "expression": {
      "op": "if",
      "condition": {
        "op": ">=",
        "left": { "field": "risk_score" },
        "right": { "value": 80 }
      },
      "then": { "value": "High" },
      "else": { "value": "Normal" }
    }
  }],
  "metrics": [{
    "key": "high_risk_count",
    "aggregation": "countWhere",
    "where": {
      "op": "=",
      "left": { "field": "risk_band" },
      "right": { "value": "High" }
    }
  }]
}
i
Divide-by-zero and invalid date operations return null instead of crashing the visual. Calculated values can feed metrics, charts, tables, maps, filters, and templates.

Maps, provider profiles, and geocoding

Security model

HyperPBI treats AI output as data, not code. The trusted-author modes broaden the accepted HTML/CSS vocabulary for controlled internal use without enabling scripts or event handlers.

Open security guide

Certification-oriented defaults

cssMode:"scoped", htmlMode:"sanitized", and showSanitizerWarnings:false.

Trusted author mode

Broader parsed declarations and HTML/SVG vocabulary for controlled dashboards—still no script execution, inline handlers, iframes, or executable CSS.

HTML

DOMPurify runs after template substitution for custom content, slots, popups, and tooltips.

CSS

CSS Tree parses rules. Selectors are scoped; imports, URLs, expressions, fixed positioning, and abusive z-index are blocked.

Network

External requests exist only in the maps build and only for configured providers on declared WebAccess domains.

!
Blocked in every mode: eval, new Function, scripts, inline event handlers, iframes, object/embed, unsafe links, external runtime scripts, and executable CSS.

Security and provider policy

Flexible enough for AI. Constrained enough for enterprise use.

HyperPBI is a renderer and compiler—not a JavaScript sandbox, iframe host, or arbitrary website runner.

Hard safety boundaries

  • ✓No eval, new Function, user functions, or inline event handlers.
  • ✓HTML, slots, templates, and map popups are sanitized with DOMPurify.
  • ✓CSS is parsed, allowlisted, and scoped to the visual or component instance.
  • ✓Scripts, iframes, object/embed, CSS imports, unsafe URLs, fixed positioning, and z-index abuse are blocked.
  • ✓AI prompts remain local until the user explicitly copies them to an AI service.
  • ✓Address geocoding is user-triggered, sequential, rate-limited, cancellable, and cached.

Do users need to write JSON?

No. The default workflow asks for a dashboard goal, generates a complete AI prompt, and imports the returned response. JSON and engine tools are grouped under Advanced.

Can it interact with other Power BI visuals?

Yes, where table selection identities, model relationships, host capabilities, and Power BI report interaction settings allow it. Internal filtering and external report selections are reported separately.

Can AI execute code inside Power BI?

No. AI output is data. HyperPBI validates JSON, evaluates a constrained calculation DSL, sanitizes HTML, parses and scopes CSS, and never executes user JavaScript.

Does it require a server?

No. The visual and authoring Studio run inside Power BI. AI prompts are copied by the user to their chosen service; HyperPBI has no AI server dependency.
