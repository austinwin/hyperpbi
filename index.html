Security and provider policy

Flexible enough for AI. Constrained enough for enterprise use.

HyperPBI is a renderer and compiler—not a JavaScript sandbox, iframe host, or arbitrary website runner.

Hard safety boundaries

  • ✓No eval, new Function, user functions, or inline event handlers.
  • ✓HTML, slots, templates, and map popups are sanitized with DOMPurify.
  • ✓CSS is parsed, allowlisted, and scoped to the visual or component instance.
  • ✓Scripts, iframes, object/embed, CSS imports, unsafe URLs, fixed positioning, and z-index abuse are blocked.
  • ✓AI prompts remain local until the user explicitly copies them to an AI service.
  • ✓Address geocoding is user-triggered, sequential, rate-limited, cancellable, and cached.

Do users need to write JSON?

No. The default workflow asks for a dashboard goal, generates a complete AI prompt, and imports the returned response. JSON and engine tools are grouped under Advanced.

Can it interact with other Power BI visuals?

Yes, where table selection identities, model relationships, host capabilities, and Power BI report interaction settings allow it. Internal filtering and external report selections are reported separately.

Can AI execute code inside Power BI?

No. AI output is data. HyperPBI validates JSON, evaluates a constrained calculation DSL, sanitizes HTML, parses and scopes CSS, and never executes user JavaScript.

Does it require a server?

No. The visual and authoring Studio run inside Power BI. AI prompts are copied by the user to their chosen service; HyperPBI has no AI server dependency.
